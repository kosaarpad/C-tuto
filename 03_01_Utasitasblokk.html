Utasításblokk

Az utasításblokk (másik nevén: szekvencia) egy olyan utasítás-sorozat, amelyet a gép a felsorolt sorrendben hajt végre. Az utasításblokkot mindig kapcsos zárójelek közé tesszük: 

{ utasítás1; utasítás2; utasítás3; utasítás4; … utasításN;}

A gép ezt balról jobbra fogja végrehajtani, ahogyan egymás után következnek.
Azonban programozástechnikai és szövegszerkesztési okokból az egymás után következő utasításokat egymás alá szoktuk írni, a jobb olvashatóság miatt, így: 

{ 
	utasítás1; 
	utasítás2; 
	utasítás3; 
	utasítás4; 
	… 
	utasításN;
}

A gép ezt fentről lefele, ugyanúgy fogja végrehajtani, mint a sorban írt verziót.
A gépnek mindegy, hogy egy sorban vagy egymás után írunk egy utasításblokkot, nekünk azonban nem, mert az utóbbit jobban át lehet tekinteni.

FONTOS!

Lehetőségünk van arra, hogy bármelyik utasításblokkban változókat deklaráljunk.
A C# nyelvben, ha egy változót az adott utasításblokkon belül deklarálunk, akkor az adott változó csak onnan látható, a blokkon kívülről már nem!
Példa:

	
using System;
class Program
{
   static void Main()
    {
    //létrehozunk egy blokkot.
	//Itt nincs értelme, csupán demo...
		{
		int i;
		i=5;
		Console.WriteLine(""+i);
		}
    }
}

A program így lefut, mert a kiíró utasítás egy blokkban van a deklarált i változóval.
De: 

	
using System;
class Program
{
    static void Main()
    {
    //létrehozunk egy blokkot.
	//Itt nincs értelme, csupán demo...
		{
		int i;
		i=5;
		}
	Console.WriteLine(""+i);	
    }
}

Nos, ez a program már le se fordul, hibaüzenetet kapunk már a fordításkor.
Ennek az az oka, hogy az i változót a blokkban deklaráltuk, azonban a blokkon kívülről akarjuk kiíratni.
A blokkon belüli változót lokális változónak nevezzük, ez a változó a blokkon kívülről nem "látszik".
