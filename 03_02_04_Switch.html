Többszörös elágazások: a switch..case utasítás

Az egymásba ágyazott if…else utasítások olykor nehezen átláthatók.
Ezért alkották meg a switch utasítást ami átláthatóbb többszörös elágazást tesz lehetővé. Szerkezete: 

	
switch (szelektor)
{
case érték1:
        utasítás1;
        utasítás2;
        break;
case érték2:
        utasítás3;
        utasítás4;
        break;
case érték3:
        utasítás5;
        utasítás6;
        utasítás7;
        break;
default:
        utasítás8;
        utasítás9;
}

Itt a szelektor egy olyan változó, amely lehet valamilyen egész típus, karakter vagy felsorolás típus. Olvasata: 

	
Ágazz el szelektor értéke alapján a következőképpen:
    Ha a szelektor = érték1-gyel, akkor hajtsd végre utasítás1-et, majd utasítás2-t!
    Ha a szelektor = érték2-vel, akkor hajtsd végre utasítás3-at, majd utasítás4-et!
    Ha a szelektor = érték3-mal, akkor hajtsd végre utasítás5-öt, utasítás6-ot, majd utasítás7-et!
    Ha a szelektor = alapértékkel, akkor  hajtsd végre utasítás8-at, majd utasítás9-et!

A break utasítás azért van, hogy a program az adott ág végrehajtása után kiugorjon a switch blokkból.
Ha a szelektor értéke érték1, akkor az utasítás1 és utasítás2 végrehajtása után kiugrik, és így az alatta lévő utasítások nem hajtódnak végre.
Például: 

	
using System;
class Program
{
	static void Main()
	{
	int jegy;
	Console.WriteLine("Hányas lett a programozás dogád?");
	jegy = Convert.ToInt32(Console.ReadLine());
	switch(jegy)
		{
		case 1:
		Console.WriteLine("Ez elégtelen!");	
		break;
		case 2:
		Console.WriteLine("Ez elégséges!");	
		break;
		case 3:
		Console.WriteLine("Ez közepes!");	
		break;
		case 4:
		Console.WriteLine("Ez jó!");	
		break;
		case 5:
		Console.WriteLine("Ez jeles!");	
		break;
		}	
	}
}

Az előző fejezetben a testtömegindex feladatnál ez az elágazás nem alkalmazható, mert a tti változó lenne itt a szelektor, de a szelektor nem lehet valós típusú.
Ha a fenti példában a case 1: utáni break; parancsot kifelejtjük, akkor az 1-esre kiírja az "Ez elégtelen!", majd az "Ez elégséges!" üzenetet is, ami itt hiba lenne.
A case belépési pont után csak egy érték adható meg, intervallum vagy több érték megadása nem megengedett, hiszen ezek az 'értékek' gyakorlatilag egy címke szerepét töltik be. Ha két értékhez rendeljük ugyanazt a tevékenységet, akkor két címkét kell definiálni.
