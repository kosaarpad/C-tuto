A destruktor

A destruktor az osztálynak az a tagfüggvénye, amely akkor hívódik meg, ha az osztály a rendszer számára már nem elérhető, nem látható.
Ez akkor történik meg, ha egy osztályt pl valamilyen lokális vátozóként hoztunk létre egy függvényben.
A függvény lefutása után erre a lokális osztályra nincs már szükség.
Ilyenkor hívódik meg a dstruktor, amelynek hatására az objektum törlődik a memóriából.
A destruktort általában nem szoktuk lekódolni, mert az automatikusan létrejön. Ha mégis lekódoljuk, akkor a destruktor neve megegyezik az osztály nevével, előtte egy "~" jellel. Példa: 

using System;
	class Vonal {
		private double hossz;   // Length of a line
		
		public Line() {   // constructor
				 Console.WriteLine("Vonel objektum létrehozása");
			}
		~Line() {   //destructor
				 Console.WriteLine("A vonal objektum törlődik");
			}
		public void setHossz( double ujhossz ) {
				 this.hossz = ujhossz;
			}
		public double getHossz() {
				 return this.hossz;
			}
		}
	class Program
	{

	static void Main(string[] args) {
			 Vonal v = new Vonal();

			 // hossz megadása
			 v.setHossz(6.0);
			 Console.WriteLine("Vonal hossza : {0}", v.getHossz());           
		}
	}

Output:


C:\Csharp>
Vonal hossza: 6.0
A vonal objektum törlődik
